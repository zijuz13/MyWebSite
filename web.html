An interesting learning experience

Never Say Never!

Currently, I am working at Huawei Company. When I started working here, I found that the technologies being used were totally different from what were used in ordinary companies. The business and service of Huawei needed to accommodate high volumes of users so that multi-threaded programming is necessary. However, I did not have any relevant experience before. After going through some articles online, the issues regarding multi-threaded programming and resolutions like CAS and synchronization still confounded me. However, I lived by the motto ‘Never say never’ and, with my great passion for computer science, I decided to get to the bottom of how these things were implemented. First, I opened the IDEA, and read the source code of Thread and Atomic classes carefully. Then, I used a conditional breakpoint to suspend the program upon reaching the line of code where the breakpoint was set. After repeating these several times, I got a rough idea of how they worked. Nevertheless, another problem presented itself. Some functions like compareAndSet in the source code were prefixed by ‘native’, and were all implemented by C++, which I was unfamiliar with. But instead of giving up, I spent my spare time learning C++. The C++ source code behind the Thread class also involved assembly language, which was more abstract and complicated. But I loved computer science so much that I forgot the predicament I was in and set my mind to understanding the assembly language as well. After weeks of hard work, I finally realized it was the 'lock cmpxchg' command which mattered. And this command took an important role in the implementation of Java Synchronization, Lock and CAS. However, I did not become satisfied with these findings. Instead, I started to read papers about Java Hotspot implementation, learning more interesting topics along the way. For example, there were two different types of locks, spin-locks and suspend-locks, and threads locking a spin-lock will loop until the lock was successfully acquired. This can be simple, and performed well with low contention. However, when contention rose, the overall performance suffered, and we could disable spin-locks to improve performance. Knowing this, I became not only more proficient in the application of multi-threaded programming, but also more fascinated by CS.
